Mossrake Language Model Diligence - Assessment

This assessment evaluates a single AI model deployed in a defined process. For each AI model in your organization's portfolio, a separate assessment should be completed by the person or team with direct knowledge of both the technical implementation and the business context.

Processes that involve multiple models, pipelines, or agentic architectures require a facilitated assessment – contact Mossrake for details.

Example descriptions

"We use a large language model to generate draft responses to inbound customer support tickets. It receives the customer's message and their account history, and generates a draft response that goes into a queue for agent review before sending."

"A language model reviews incoming invoices against purchase orders and flags discrepancies. Flagged items go to accounts payable for manual review; matched items are auto-approved for payment up to $5,000."

"Our compliance team uses a language model to screen marketing copy against regulatory guidelines before publication. The model identifies potential violations and generates a summary for the compliance officer to review."

Q1 How much does it matter if this model produces a wrong output? Consequence

This determines how the remaining questions are evaluated.

Formal diligence may not be necessary for this process.

At consequence level 1, the stakes may not justify a structured assessment. Consider revisiting if the model's role changes or if downstream consequences become clearer.

Q2 Do you know exactly which model and version is running this process? Identity

Describe how the model is specified in your implementation. If you use a framework, wrapper, or routing layer, note that here.

Q3 If the model or its behavior changed, would you know? Drift

Describe how you would learn that this model's behavior has changed. Include any monitoring, testing, or alerting mechanisms in place.

Q4 Is output quality checked before the output reaches its end use? Verification

Describe what happens to the model's output between generation and end use. Who or what checks it, and what happens when an output is flagged?

Q5 If a bad output got through, how quickly would it be caught? Recovery

Describe how a bad output that passed any existing checks would eventually come to someone's attention.
